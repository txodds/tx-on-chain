The TxLINE scores feed delivers granular sports score data that can be validated against on-chain Merkle roots. Current documentation covers soccer, US football, and basketball score encodings, with World Cup and International Friendlies coverage available through the free tiers where enabled.

For NCAA Football and NCAA Basketball, TxLINE collects play-by-play data in venue and cross-verifies it with US-based analysts. Subscriber delivery timing follows the selected service level; do not infer sub-second delivery for a delayed tier.

Every supported stat is encoded with deterministic keys and published through batches whose roots are anchored on Solana. Use the feed-specific pages for stat-key encoding, and use the [Schedule](/documentation/scores/schedule) and API responses to confirm current fixture availability.

For on-chain validation, use [On-Chain Validation](/documentation/examples/onchain-validation). The current examples include both legacy `validateStat` requests and `validateStatV2` multi-stat strategies through [Runnable Devnet Examples](/documentation/examples/devnet-examples).

For final match outcome settlement on the current devnet and mainnet releases, filter `action=game_finalised` separately from ordinary score updates and require `statusId=100` plus proof `period=100`. These release-specific finalisation markers are distinct from sport phase IDs; re-verify them when adopting V3 or a newer API/program release.

## Built for Settlement & Micro-Markets

This feed is purpose-built for high-frequency betting environments: in-play markets, player props, and micro-betting.

- **Deterministic Encoding:** Every stat maps to a fixed cryptographic key with period multipliers applied on top.
- **Reliable Proofs:** If you are building automated smart contract settlement logic or validation proofs, the exact layout schemas are defined in the technical specifications below.
