## Stream Expectations

An open SSE connection means your credentials and stream request are accepted. It does not guarantee that a covered fixture is producing data at that exact moment.

If the stream opens but you only see heartbeats or no data messages for a while:

- Check [Scores Schedule](/documentation/scores/schedule) for active or upcoming covered fixtures.
- Keep the stream open during a covered live fixture window.
- Use `/api/scores/historical/{fixtureId}` when you need replay data for a completed fixture within the historical availability window.
- Confirm you are using the correct network host for your subscription.

<Note>
If a stream returns `401`, renew the guest JWT once from the same host and reconnect with the same `X-Api-Token`. If it returns `403`, check the API token, subscription, network, and bundle permission. Do not renew the JWT on `403`, and bound every reconnect attempt.
</Note>

## SSE Parsing Helper

```typescript
type SseMessage = {
  id?: string;
  event?: string;
  data: string;
  retry?: number;
  comments: string[];
};

function parseSseBlock(block: string): SseMessage | null {
  const message: SseMessage = { data: "", comments: [] };

  for (const rawLine of block.split(/\r?\n/)) {
    if (!rawLine) continue;
    if (rawLine.startsWith(":")) {
      message.comments.push(rawLine.slice(1).replace(/^ /, ""));
      continue;
    }

    const separatorIndex = rawLine.indexOf(":");
    const field = separatorIndex === -1 ? rawLine : rawLine.slice(0, separatorIndex);
    const value =
      separatorIndex === -1
        ? ""
        : rawLine.slice(separatorIndex + 1).replace(/^ /, "");

    if (field === "data") message.data += `${value}\n`;
    if (field === "event") message.event = value;
    if (field === "id") message.id = value;
    if (field === "retry") message.retry = Number(value);
  }

  message.data = message.data.replace(/\n$/, "");
  return message.data || message.event || message.id || message.comments.length
    ? message
    : null;
}

async function* readSseMessages(response: Response): AsyncGenerator<SseMessage> {
  if (!response.body) throw new Error("Stream response has no body");

  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  let buffer = "";

  try {
    while (true) {
      const { value, done } = await reader.read();
      if (done) break;

      buffer += decoder.decode(value, { stream: true });

      let separator = buffer.match(/\r?\n\r?\n/);
      while (separator?.index !== undefined) {
        const block = buffer.slice(0, separator.index);
        buffer = buffer.slice(separator.index + separator[0].length);

        const message = parseSseBlock(block);
        if (message) yield message;

        separator = buffer.match(/\r?\n\r?\n/);
      }
    }

    buffer += decoder.decode();
    const message = parseSseBlock(buffer);
    if (message) yield message;
  } finally {
    reader.releaseLock();
  }
}

```

Bound smoke probes to 30–45 seconds. Classify the result instead of treating every open connection as a data success:

- An accepted response or heartbeat is an authentication and transport smoke pass.
- A data event is a data-flow pass.
- An accepted stream with no data event is `inconclusive`, because no covered fixture may be active.
- A non-`2xx` response, parser error, or timeout before the response is accepted is a failure.

```typescript
async function probeSse(streamUrl: string, seconds = 35) {
  if (seconds < 30 || seconds > 45) {
    throw new Error("SSE probe duration must be between 30 and 45 seconds");
  }

  const controller = new AbortController();
  const timer = setTimeout(() => controller.abort(), seconds * 1000);
  let accepted = false;
  let sawHeartbeat = false;
  let sawData = false;

  try {
    const response = await fetch(streamUrl, {
      signal: controller.signal,
      headers: {
        Authorization: `Bearer ${jwt}`,
        "X-Api-Token": apiToken,
        Accept: "text/event-stream",
        "Cache-Control": "no-cache",
      },
    });

    if (!response.ok) {
      await response.body?.cancel().catch(() => undefined);
      throw new Error(`Stream failed: ${response.status}`);
    }
    const contentType = response.headers.get("content-type") ?? "";
    if (!contentType.toLowerCase().includes("text/event-stream")) {
      await response.body?.cancel().catch(() => undefined);
      throw new Error("Stream returned an unexpected content type");
    }
    accepted = true;

    for await (const message of readSseMessages(response)) {
      const namedHeartbeat =
        message.event === "heartbeat" || message.event === "ping";
      if (namedHeartbeat) {
        sawHeartbeat = true;
      } else if (message.data) {
        sawData = true;
        console.log("SSE data event", {
          id: message.id,
          event: message.event ?? "message",
          bytes: new TextEncoder().encode(message.data).length,
        });
      } else if (message.comments.length > 0) {
        sawHeartbeat = true;
      }
    }
  } catch (error) {
    if (!(error instanceof Error && error.name === "AbortError")) throw error;
  } finally {
    clearTimeout(timer);
  }

  if (sawData) return "data-flow-pass" as const;
  if (accepted || sawHeartbeat) return "transport-pass-data-inconclusive" as const;
  throw new Error("SSE probe timed out before the stream was accepted");
}
```

## Historical Scores

Fetch the complete sequence of score updates for a fixture that started between two weeks and six hours ago.

```typescript
import axios from "axios";

const httpClient = axios.create({
  timeout: 30000,
  headers: {
    "Content-Type": "application/json",
    "Authorization": `Bearer ${jwt}`,
    "X-Api-Token": apiToken
  },
  baseURL: "https://txline.txodds.com",
});

const fixtureId = Number(process.env.TXLINE_FIXTURE_ID);
if (!Number.isSafeInteger(fixtureId) || fixtureId <= 0) {
  throw new Error("Set TXLINE_FIXTURE_ID to an observed fixture ID");
}

const historicalScores = await httpClient.get(`/api/scores/historical/${fixtureId}`);

console.log(`Retrieved ${historicalScores.data.length} score updates for fixture ${fixtureId}`);
historicalScores.data.forEach((update, index) => {
  const seq = update.Seq ?? update.seq;
  const ts = update.Ts ?? update.ts;
  const fixture = update.FixtureId ?? update.fixtureId;
  console.log(`${index + 1}. Fixture: ${fixture}, Seq: ${seq}, TS: ${ts}`);
});
```

Use the observed sequence from a historical record when requesting a validation proof for that record.

<Info>
**Historical Availability**: This endpoint only returns data for fixtures with start times between two weeks and six hours in the past from the current time.
</Info>

<Info>
**Stream Compression**: Let native `fetch`/Undici negotiate and decompress the response. Do not set `Accept-Encoding` manually, and do not pass already-decoded chunks to `gunzipSync`. For command-line probes, use `curl --compressed`. If a lower-level client exposes a genuinely raw gzip body, pipe the complete byte stream through `createGunzip` before SSE decoding; a gzip frame can span transport chunks.
</Info>
